### nba.boxscores_advanced

- game_id text REFERENCES nba.games(game_id)
- nba_id integer REFERENCES nba.players(nba_id)
- team_id integer REFERENCES nba.teams(team_id)
- minutes numeric(8,2)
- off_rating numeric(6,2)
- def_rating numeric(6,2)
- net_rating numeric(6,2)
- ast_pct numeric(5,4)
- ast_to_ratio numeric(6,2)
- ast_ratio numeric(6,2)
- oreb_pct numeric(5,4)
- dreb_pct numeric(5,4)
- reb_pct numeric(5,4)
- tm_tov_pct numeric(5,4)
- efg_pct numeric(5,4)
- ts_pct numeric(5,4)
- usg_pct numeric(5,4)
- e_off_rating numeric(6,2)
- e_def_rating numeric(6,2)
- e_net_rating numeric(6,2)
- e_ast_ratio numeric(6,2)
- e_oreb_pct numeric(5,4)
- e_dreb_pct numeric(5,4)
- e_reb_pct numeric(5,4)
- e_tm_tov_pct numeric(5,4)
- e_usg_pct numeric(5,4)
- e_pace numeric(6,2)
- pace numeric(6,2)
- pace_per40 numeric(6,2)
- poss integer
- pie numeric(5,4)

- created_at timestamptz
- updated_at timestamptz
- fetched_at timestamptz

#### Constraints & Indexes
- PRIMARY KEY (game_id, nba_id)
- INDEX ON (team_id)
- INDEX ON (nba_id)
- INDEX ON (game_id)

#### Notes
- `game_id` is the official 10-digit NBA game identifier.
- `nba_id` is the official player identifier.
- `minutes`: Stored as decimal minutes (`numeric(8,2)`).
- Percentages (e.g., `ast_pct`, `ts_pct`) are stored as `numeric(5,4)` assuming the API provides them as decimals (e.g., 0.5432).
- Ratings (e.g., `off_rating`, `def_rating`) are stored as `numeric(6,2)` to support standard "per 100 possessions" values (e.g., 115.42).
- `e_*` columns refer to "Estimated" metrics often found in NBA advanced stats.
- `poss`: Total possessions for the player while on the court.
- `pie`: Player Impact Estimate.
- Measure Type: Advanced (per `nba-stats.txt`).
- This table tracks the advanced performance metrics per player per game.
